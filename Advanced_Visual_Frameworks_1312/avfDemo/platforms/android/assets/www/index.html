Data Research and Project Intent

1. Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios?

The average user today can have quite an extensive list of logins and passwords. This can be simplified by integrating optional automatic sign-ins from companies that users already have. Sign in with Twitter is available on websites and mobile and desktop applications. Not only does this provide the user with a two-click sign-in, the users now the ability to integrate Twitter APIÍs through the app. American Express has teamed up with Twitter to offer payments through the use of a hashtag tweet. Although for this to be used the user need a compatible American Express card and a cooperating product.

2. Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?

JSON is a simple data-interchange format, it is bounded by the same-origin policy. The policy essentially prevents the pages from getting data from any page that does not reside within itÍs main domain name as a way to protect users from unwanted or obtrusive acts. JSON-P, while still being a JSON format, works around the same-origin policy by using JavaScript to pull specific data.

3. Based on the JSON-P search results: what data resource(s) do you think might pair well with which types of Native device features?

There are some Native device features that work great with specific APIÍs. A mapping API like Google Maps would probably pair best with Geolocation, Flicker would work well with the camera and social APIÍs like Twitter and Facebook would pair well with the camera.

4. Please describe any design and functionality intentions you may have for your Demo app components this month.

One of my previous apps was a biking app. With that theme I was able to come up with many possible ways to combine for my demo app. I could use Geolocation and maps to track rides, post those times and distances to twitter, post camera images to twitter, or pull latest tweets or Instagram images to compare with. I could also pull weather data to determine specific trail conditions.

5. Briefly describe some past 'lessons learned' regarding workflow as it relates to research, design, development, etc.

Previously the area where I have had the most difficulty is when I try to see the end in the beginning. Due to this, I end up reworking my project repeatedly. I have learned that I need to develop a clear vision and lay out my project only after thorough research. While I donÍt mind reworking my project, since I most likely learned something in the process, I would like to reduce unnecessary work.
